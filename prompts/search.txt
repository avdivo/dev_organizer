USER:
Нужно сформировать фильтр по метаданным по запросу пользователя.
И учитывая контекст запроса определить способ выборки и обработки данных.
Ты, как эксперт, должен четко понять что нужно пользователю и
предложить оптимальный вариант поиска.
1. В поле "search" укажи одну из опций, после проведения анализа и понимания запроса
   - "llm_smart" - если нужно произвести арифметические расчеты, ответить на сложный вопрос
   - "llm_lite" - ответит на большинство вопросов, посчитает количество записей, не нужна для отбора и прочтения
   - "filter" - умеет быстро фильтровать записи и возвращает их как есть
   - "semantic_and_llm" - произвести семантический поиск в БД и вернуть ответ через llm
2. Даты и время — в ISO 8601. Может быть указана одна дата или промежуток:
   "вчера", "за прошлый месяц", в "за эту минуту" - найти начало и конец промежутка.
   "до сегодня" - до 00:00:00 сегодняшнего дня, "до сейчас" - до текущего времени
3. Если фильтр не уточняется, поле опустить.

Сформируй фильтр для ChromaDB в JSON следующей структуры:
{
    "filters": [
        {"datetime_create": {"$gte": "2025-06-01T00:00:00"}}, дата создания ISO 8601
        {"datetime_create": {"$lte": "2025-06-01T00:00:00"}},
            или
        {"datetime_reminder": {"$gte": "2025-06-01T00:00:00"}}, дата напоминания ISO 8601
        {"datetime_reminder": {"$lte": "2025-06-01T00:00:00"}},
    ],

    "search": "...",
    "essence": только суть поисковой фразы для семантического поиска
    "where_document": ... искомое слово или фраза, о нем прямо должны быть сказано:
    "найди слово шнур", "со словом вода", "где есть слово паста" иначе опустить поле
}
Никаких дополнительных комментариев, только json.
