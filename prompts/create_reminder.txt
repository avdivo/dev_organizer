USER:
1. Создай список напоминаний. Соблюдай правила:
   - если упоминается несколько дат, создай отдельное напоминание для каждой
   - сохраняй смысл напоминания в каждом
   - добавляй в каждое напоминание дату (если она есть)
   - если напоминание периодическое добавь период напоминания.
2. Единицы измерения полным текстом, если не указана подбери по контексту фразы.
   Преобразуй единицу измерения в каноническую форму —
   в именительном падеже, в единственном числе, без сокращений и спецсимволов.
   Правила:
   - по умолчанию для денег применяй рубли
   - люди, животные, предметы, действия, пачки, упаковки → штуки
   - номер (любой номер, сущности, порядковый) → номер
   - баллы, рейтинг, оценка → бал

3. Верни это в виде list[dict]:
[
    {
        "data":
            {
                "text": напоминание без служебных слов о напоминании (фраза должна отвечать на вопрос "что сделать?")
                "datetime_create": создания заметки  (обязательное поле) ISO 8601
                "datetime_reminder": когда напомнить первый раз? время сработки прошло? СТАВЬ СЛЕДУЮЩЕЕ!

                "numbers": [{5 (int/float): "километр в час"}] могут повторяться
            },
        "APScheduler": {

            },
        "answer": - текст напоминания, режим повторов (если есть),
                    дата и время первого напоминания (в соответствии с правилами).

    }
 ]

 4. Для напоминаний:
- для словаря APScheduler используй формат даты ISO 8601
- если время не указано ставь 00:00:00
- вычисляй дату и время первого напоминания в будущем. Если время сработки прошло? СТАВЬ СЛЕДУЮЩЕЕ!
- для понятий завтра. в понедельник, через месяц и др. вычисляй дату и записывай в
  "datetime_reminder" (поле обязательное)
- для периодов следующий месяц, следующую неделю и др. ставь дату начала периода

Примеры напоминаний для APScheduler:
{
  "trigger": "date",
  "run_date": "2025-06-17T18:00:00+03:00"
}

{
  "trigger": "cron",
  "month": "1",
  "day": "1",
  "hour": "0",
  "minute": "0"
}

5. Никаких дополнительных комментариев, только описанная структура.
6. Если запрос не пустой ты должен вернуть как минимум одну запись с обязательными полями.
